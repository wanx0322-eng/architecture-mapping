# 可审计进化治理

## 触发周期

维护两个独立计数器：

- 每累计10次任务对话，触发一次“对话适配复盘”；
- 每累计生成10张图片，触发一次“图像稳定性复盘”；
- 连续3次同类错误可提前触发 `repair-only` 复盘。

一次任务只计一次对话事件；一次工具调用生成多张图片时按实际图片数累计。复盘成功后只重置
对应计数器，不清空历史总数。

## 进化流程

1. **Observe**：读取最近事件、路由、分数、失败、用户反馈和返工次数。
2. **Diagnose**：把问题归为路由错误、证据缺失、提示词漂移、图底失败、风格失控、
   中文排版、工具故障或评测缺口。
3. **Propose**：提出 Gene/规则、参考文档、模板、Schema、脚本或评测修改。
4. **Review**：使用 Capability Evolver 的 review 模式；外部建议不能直接覆盖本地文件。
5. **Test**：运行受影响单元测试、Schema、代表任务和静态评测查看器。
6. **Approve**：新增/修改在测试通过后可固化；删除必须单独获得用户批准。
7. **Solidify**：更新版本、变更日志和审计事件；失败时恢复到基线包。
8. **Monitor**：观察下一个10次周期是否真正减少返工和漂移。

## Capability Evolver 安全配置

- 使用本地 Proxy 邮箱，不直接调用 Hub API。
- 首选 `--review`；保持 `EVOLVE_ALLOW_SELF_MODIFY=false`。
- 禁止 `hard` 回滚；使用可恢复快照或 stash 思想，但不得覆盖用户未提交改动。
- 向外部发送前移除用户路径、图片、项目名称和隐私数据，只发送抽象错误信号与指标。
- Proxy、节点或环境变量缺失时，执行本地复盘，不尝试绕过。

## 删除与弃用

删除字段、Schema属性、参考文档、脚本或模板之前，提案必须包含：

1. 精确对象和当前用途；
2. 引用与依赖扫描结果；
3. 删除依据和至少一个真实失败案例；
4. 替代项及迁移方式；
5. 对历史数据和旧提示词的兼容影响；
6. 回滚文件或基线包；
7. 用户明确批准记录。

默认处理是标记 `deprecated`，保留至少两个进化周期。未经批准不得删除。即使用户批准，也只能
一次处理一个明确路径或字段，不得批量删除目录或字段集合。

## 成功指标

- 首轮方案通过质量阈值的比例；
- 平均返工次数；
- F2/F3几何违规数；
- 图底层级和缩略图可读性评分；
- 中文文字错误率；
- 用户主动修改提示词的次数；
- 相同输入多次生成的结构一致性。
