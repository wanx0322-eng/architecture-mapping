# MoE 专家路由与仲裁

## 总体原则

总路由负责理解任务、拆解、选择专家、合并结果和控制状态；子专家只处理自己边界内的工作。
“混合专家”不是多数投票，而是相关专家提出可核验意见，由硬约束和质量评分共同决定。

## 路由阶段

1. **任务识别**：分析、分类、数据集、提示词、出图、编辑、排版、质量复核或进化。
2. **门槛检查**：涉及制作时，先调用简报专家；3轮之前阻止模型出图专家。
3. **专家选择**：选1个主专家、2至4个支持专家；质量专家强制参与最终交付。
4. **子任务契约**：每个专家得到输入、预期输出、硬约束和完成条件。
5. **候选合成**：专家可提出独立方案或对同一方案评分。
6. **MoE仲裁**：先执行硬否决，再计算加权总分。
7. **执行与复核**：模型出图后由质量专家复评，最多两轮针对性修正。
8. **事件记录**：记录路由、得分、错误、用户反馈和图片数量。

## 评分与硬否决

质量总分为100：

- 证据与几何忠实度25；
- 分析逻辑20；
- 图底与视觉层级20；
- 缩略图/打印可读性15；
- 艺术风格一致性10；
- 生产与编辑就绪度10。

以下情况直接否决：虚构数据、F2/F3几何漂移、中文乱码作为终稿、核心结论不可读、
违反用户明确禁止项、制作问询不足3轮、未经批准执行删除。

总分低于80返工；80–89可交付但列明风险；90以上为优选。最高两案差值小于5分或
专家置信度低于0.7时，向用户说明取舍并请求选择。

## 故障退化

- 单个专家失败：保留其错误，使用同领域支持专家提供保守方案，不隐瞒缺口。
- 图像模型失败：不反复随机重生；回到最后一个通过质量检查的版本并缩小修改范围。
- 数据不足：降低到F0/F1，标注概念表达；不伪装成精确分析。
- 外部进化服务不可用：继续本地记录和评测，不阻塞正常出图。
