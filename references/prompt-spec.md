# 提示词编译规范

## 通用 JSON

通用提示词必须依次描述：输入基底、忠实度等级、不可改变项、主类与子型、图纸形态、
分析目标、证据链、细节等级、图底关系、投影与视角、版式、色板、线型、符号、
艺术媒介、文本系统、禁止事项和输出规格。

生成提示词也属于“制作图纸”，必须先按 `interaction-protocol.md` 完成 3–7 轮问询，
并形成通过 `production_brief.schema.json` 校验的制作简报。

## 中文文字策略

- 图像模型只生成短标题、编号和极短标签；长说明留空位。
- 中文说明由 SVG/PPTX 后期排版，使用无衬线字体和可编辑文本框。
- 不允许把模型生成的乱码当作最终文字。

## Nano Banana / Gemini

- 使用结构化 JSON，明确元素位置、层级、视觉锚点和禁止项。
- 多步骤图保持同一视角、同一尺度、同一建筑身份。
- 参考图存在时优先强调形体与道路边界忠实度。
- 把背景、基底、主题、标注和文字的层级位置分别写清楚。

## GPT Image

- 把输入图像角色、保留项与编辑项分开写。
- 明确“不要改变场地实体和建筑比例”。
- 先生成无长文本版本，再进行确定性中文排版。
- 对图底关系、边缘质量、纹理、阴影和合成方式使用独立描述，不使用笼统“高级感”。

## Midjourney

- 输出紧凑自然语言提示词和参数建议。
- 不宣称能精确复现中文正文或真实 GIS 数据。
- 建议使用参考图权重、固定比例和较低风格化强度。
## Pinterest 数据集风格编译

仅在 `style_source=pinterest_dataset` 时启用。读取 `pinterest_style_profiles.json`，把簇特征拆成媒介、边缘、纹理、阴影、色板、图底、版式密度和反向词。`style_strength` 编译为 low / medium / high 三档，但任何强度都不得覆盖 F2/F3 几何、证据链、锁定道路、比例或真实 GIS 数据。

- Gemini：把风格写入结构化 `style_reference`，逐项声明保留项和禁止项。
- GPT Image：把输入角色、不可编辑几何、可编辑视觉层和 negative constraints 分段。
- Midjourney：压缩为可观察视觉短语，降低 stylize，禁止声称精确复刻文字或数据。
- 不写笼统“Pinterest 风”“高级感”；不得复制任何代表 Pin 的场地、道路、文字、功能、数据或设计方案。
